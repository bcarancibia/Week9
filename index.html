Value Added - Week 9 CUNY Bridge Courses

The assignment for the Week 9 of the CUNY Bridge course was to take a dataset provided to the class and work as a team to perform an exploratory data analysis that compares the four datasets.

The dataset we had to use can be found here.

The exploratory data analysis proceed in an interesting manner that I don't think was expected by the administrators of the assignment. The data analysis began by graphing the data set and immediately recognizing that the dataset was very similar to the classical Anscombe's Quartet. Graphing the dataset was done in R and as noted in the discussion Anscombe's Quartet shows the failings of regression lines.

The analysis of this data was performed using R and Python with specific packages that focused on graphing. These packages are were ggplot2 for R and matplotlib for Python.

The first set of analysis done on the data were summary statistics, it was discovered that the datasets all had very similar mean, variances, maximums, minimums, and similar linear regression. Some of the key points that the group found while performing this exercise is that diagnostic checks should be run to verify the model. Finding the biggest influences on the model can be useful for diagnosing problems with a model.

Checking for influencing data points found that in each dataset there were one or two outliers. The analysis from the group found that a simple linear regression model is the way to start, but that model should be tested and alternative models should be considered. Analysis must verify that the model is a good fit and that the assumptions behind the linear model are satisfied.

Some comments on R and Python

Both R and Python provide functionality for reading a .csv and R provides a native way to load data from a .csv file. Python does not provide methods for loading data into a data frame. Users can integrate pandas, numpy, or scipy to load the .csv file into a dataframe within Python. R can create plots natively and again Python needs to have a package downloaded and imported before you start using Python. The plots are quite similar.

The group came to the conclusion that as a language specifically for statistical analysis, R outperforms Python in the areas of data manipulation and exploratory data analysis.

I disagree with this group opinion. R is good language for purely statistical programming, but that is the limit of R. R is purely a language for statistical analysis. Python allows for a wide variety of options besides just data manipulation and statistical analysis. Python with it's vast libraries allows a user to do more activities and much quicker than just purely R.

Anscombe's Quartet - Real World Example

There are several real world examples of datasets that have some of the key issues raised in Anscombe's Quartet. One of these examples of the distribution of starting salaries for new law graduates.

In 2012 the National Association of Law Placement (NALP) reported that lawyers made on average $80,798 on average in starting salary, but when you look at the salary distribution you see that the law salaries fall into two camps.

Graphic found here

The majority of new lawyers make somewhere between $35,000 and $75,000 per year and a sizable minority earns $160,000 per year. The average the NALP saw falls in the trough between the two peaks, and few lawyers actually have salaries near that number. Also, something else that was discovered when statisticians graphed the data is there is a giant speak at exactly $160,000 rather than a peak with some variance. It turns out the salaries the NALP display are not actual legal salaries, but the NALP display data that law schools report to the NALP. Visualizing the data allowed users to get a better picture of what realistic salaries are and expose a potential flaw in the NALP data.
